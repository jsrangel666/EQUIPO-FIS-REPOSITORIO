# REQUERIMIENTOS
## Introducción
*LUMEN es una aplicación interactiva orientada a estudiantes, diseñada para la organización visual de ideas, metas y procesos creativos–académicos. 
El sistema prioriza la experiencia del usuario y la introspección guiada, integrando elementos visuales y no tradicionales de interacción.*

## Objetivos del sistema
*Desarrollar una aplicación que permita al usuario:*

- Organizar su semestre o proyectos personales
- Visualizar su pensamiento de forma gráfica
- Registrar su proceso creativo y académico
- Mantener una bitácora evolutiva de su crecimiento

## Usuarios
- **Usuario principal:** Estudiante universitario.

- **Usuario futuro:** Cualquier persona interesada en organización creativa, reflexión y planeación personal.

# REQUERIMIENTOS FUNCIONALES

- **RF-01:** El sistema debe permitir al usuario crear una identidad mediante un gesto dibujado.

- **RF-02:** El sistema debe validar el gesto para permitir el acceso.

- **RF-03:** El sistema debe mostrar un mapa visual interactivo.

- **RF-04:** El usuario debe poder crear nodos en el mapa.

- **RF-05:** El usuario debe poder editar y eliminar nodos.

- **RF-06:** El usuario debe poder conectar nodos entre sí.

- **RF-07:** Cada nodo debe permitir agregar texto descriptivo.

- **RF-08:** El sistema debe guardar la información del usuario.

- **RF-09:** El sistema debe permitir cargar información previamente guardada.

# REQUERIMIENTOS NO FUNCIONALES

- **RnF-01:** La interfaz debe ser intuitiva y visualmente clara.

- **RNF-02:** El sistema debe responder en tiempo real a las interacciones.

- **RNF-03:** El diseño debe priorizar la experiencia del usuario (UX).

- **RNF-04:** El sistema debe ser modular y escalable.

## RESTRICCIONES

- El sistema se desarrollará con tecnologías web.

- La persistencia se realizará de manera local.

- El proyecto se desarrollará bajo fines académicos.
